خب راستشو بخوای، بیشتر از این خسته شدم که همه فکر می‌کنن React تنها راهه.

من یه مدته دارم با Web Components کار می‌کنم و واقعاً بهش باور دارم. همون آدم غرغرویی‌ام که همیشه می‌گه: «وقتی خود مرورگر همه چی رو می‌ده، چرا باید بریم سمت کتابخونه؟»

تجربه‌ام از HTML2 شروع نشده ولی با نسل جدید HTML5، و قابلیت‌هایی مثل Custom Elements، Shadow DOM و HTML Templates، واقعاً حس می‌کنم هر چی برای ساخت یه سیستم کامپوننتی تمیز و مدرن نیاز داریم، مرورگر خودش بهمون می‌ده.

قبلاً مثل همه با React و حتی یه مدت Angular کار کردم، ولی هر چی جلوتر رفتم، بیشتر به این نتیجه رسیدم که اون لایه‌های اضافه فقط کارو پیچیده‌تر می‌کنن، نه بهتر.

الان ۲۰۲۵ئه، ولی هنوز خیلیا بدون اینکه حتی یه‌بار Web Components رو امتحان کنن، چسبیدن به React.

## روش استاندارد

React یه زمانی غوغا کرد؛ مدل کامپوننتی و آپدیتای واکنشی‌اش یه تکون حسابی به فرانت داد. ولی وابستگی به JSX، سیستم رویداد مصنوعی، سربار سرعت، قاطی شدن استایل با منطق… همه‌ش یه جورایی دور از حال و هوای استانداردای وبه.

Web Components دقیقاً دل خودشون می‌گن: استاندارد. با خود مرورگر رفیقن، بهینه‌سازی‌های توی خونه دارن، و تفکیک نگرانی‌ها (Separation of Concerns) رو خودشون داد می‌زنن. نه کامپایلر خاص، نه لایبرری زمان اجرا. داری با پلتفرم حال می‌کنی، نه برعکس.

## ذاتاً همه‌جایی

خفن‌ترین ویژگی Web Components اینه که هرجا بندازیش، جا می‌گیره. دلت می‌خواد بذاریش وسط Vue، Svelte یا هر فریمورک خفنِ امسال؟ یا حتی یه صفحهٔ HTML خام؟ فرقی نمی‌کنه، کار می‌کنه.

ولی React؟ تا میای کامپوننتاتو ببری یه فریمورک دیگه، باید نصف کد رو بازنویسی کنی، وابستگیاش گره می‌خوره، آخرش هم یکی درمیون می‌شکونه. چرا خودتو زندونی یه اکوسیستم کنی وقتی خود مرورگر زبون مشترک رو داره؟

یه Custom Element خوب، کاملاً اعلامی (Declarative) کار می‌کنه، با SSR قاطی می‌شه، با CSS معمولی شیک می‌شه، خودش کپسوله‌ست و… بیخیال، لیستش تمومی نداره.

## اوضاع فعلی

نمی‌گم React بده یا به درد نخور. ابزار قدرتمندیه، پروژه‌‌های موفق هم کم نداره. ولی الان ۲۰۲۵ه؛ این‌که هنوز خیلیا بدون فکر می‌رن سراغش، بیشتر شبیه عادته تا نیاز.

برای ماهایی که سالهاست با Web Components حال می‌کنیم، پافشاری رو راه‌حل‌های وابسته به فروشنده یه کم ناموزونه. خیلی از دردایی که React قبلاً دوا می‌کرد، سال‌هاست تو خود پلتفرم حل شده. Web Components کاملاً جاافتاده‌ن. بیاید مشکلات امروزو با ابزارای دیروز حل نکنیم؛ انگار هنوز واسه چیدمان، جدول می‌چیدیم چون «همیشه همین بوده».

## ساختن برای فردا

طبیعیه آدم سراغ ابزارای آشنا بره، ولی گاهی باید یه قدم عقب بریم و ببینیم هنوز بهترین گزینه‌ان یا نه. استانداردا دقیقاً برای همینه که زنده‌ان و رشد می‌کنن. منم همیشه رو همین موج می‌مونم. چون یه روز React هم مثل jQuery می‌ره تو کما، ولی Web Components تا وقتی وب هست، هستن.
